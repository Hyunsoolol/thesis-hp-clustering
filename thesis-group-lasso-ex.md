## 참고 자료
#### 그룹 라쏘의 페널티 감소 공식

업데이트 이전의 원래 값을 $\tilde{\delta}_{\cdot k}$, 페널티 강도를 $\lambda$, 업데이트된 새로운 값을 $\hat{\delta}_{\cdot k}$라고 할 때 공식은 다음과 같습니다.

$$\hat{\delta}_{\cdot k} = \max\left(0, 1 - \frac{\lambda}{\|\tilde{\delta}_{\cdot k}\|_2}\right) \tilde{\delta}_{\cdot k}$$

이 공식이 작동하는 과정은 다음과 같습니다.

#### 1단계: 그룹의 총 에너지(Norm) 측정

먼저 해당 변수 그룹이 가진 전체 파워, 즉 $L_2$ 노름(Norm)인 $\|\tilde{\delta}_{\cdot k}\|_2$를 계산합니다. (원소들을 각각 제곱해서 더한 뒤 루트를 씌운 값입니다.)

#### 2단계: 생존 비율(Factor) 계산

공식의 괄호 안쪽인 $1 - \frac{\lambda}{|\tilde{\delta}_{\cdot k}|_2}$를 계산합니다.

- 전체 에너지 중에서 페널티($\lambda$)가 깎아먹는 비중을 계산한 뒤, 1(100%)에서 그만큼을 빼주는 것입니다.
    
- 예를 들어 전체 에너지가 `2.0`인데 페널티 $\lambda$가 `0.5`라면, $1 - (0.5 / 2.0) = \mathbf{0.75}$가 됩니다. 즉, **"이 그룹은 원래 힘의 75%만 남기고 축소시켜라"**라는 비율이 산출됩니다.
    

#### 3단계: 비율 일괄 곱하기 (Scaling) & All-or-Nothing

산출된 비율을 그룹 내의 **모든 개별 원소에 똑같이 곱해줍니다.**

- **구조 보존 (생존):** 비율이 양수라면, 군집 1, 2, 3의 값에 똑같이 0.75를 곱합니다. 똑같은 상수를 곱했기 때문에 값의 크기만 줄어들 뿐, **군집 간의 비율(상대적 거리)은 완벽하게 유지**됩니다. (파편화 방지)
    
- **All-or-Nothing (제거):** 만약 페널티 $\lambda$가 전체 에너지보다 커서 괄호 안의 값이 마이너스가 되면 어떻게 될까요? 이때 앞에 붙어있는 $\max(0, \dots)$ 함수가 작동하여 비율을 **0**으로 만들어 버립니다. 0을 곱하니까 그룹 전체가 한 번에 0으로 붕괴하여 탈락하게 됩니다.

#### 일반 라쏘와의 차이점 비교

- **일반 라쏘 (Element-wise Soft-Thresholding):**
    
$$\hat{\delta}_{jk} = \text{sign}(\tilde{\delta}_{jk}) \max(0, |\tilde{\delta}_{jk}| - \lambda)$$
    
개별 값에서 $\lambda$를 직접 **뺍니다**. 그래서 값마다 깎이는 타격이 다르고, 작은 값은 먼저 0이 되어 파편화가 발생합니다.
    
- **그룹 라쏘 (Block Soft-Thresholding):**
    
$$\hat{\delta}_{\cdot k} = \max\left(0, 1 - \frac{\lambda}{\|\tilde{\delta}_{\cdot k}\|_2}\right) \tilde{\delta}_{\cdot k}$$
    
벡터 방향은 그대로 둔 채, 길이만 비율로 **압축**합니다. 그래서 영원히 파편화가 발생하지 않습니다.
    

#### 그룹 라쏘 예시

- **원래 군집 편차 ($\tilde{\delta}_{1}$):** $(2.0, -1.0, -1.0)$
    
- **페널티 강도 ($\lambda$):** **$1.2$** 
    
##### 1단계: 그룹의 총 에너지(L2 Norm) 계산

개별 값을 보지 않고, 3개의 값을 하나의 벡터로 묶어 피타고라스의 정리처럼 총길이 계산

$$\|\tilde{\delta}_{1}\|_2 = \sqrt{2.0^2 + (-1.0)^2 + (-1.0)^2} = \sqrt{4 + 1 + 1} = \sqrt{6} \approx \mathbf{2.449}$$

##### 2단계: 생존 비율(Shrinkage Factor) 산출

그룹 전체의 에너지($2.449$)가 페널티($1.2$)보다 크기 때문에 이 그룹은 살아남습니다.
이제 **"원래 값에서 몇 %를 남길 것인가?"**를 계산하며, 공식은 다음과 같습니다.

$$\text{Factor} = \max\left(0, 1 - \frac{\lambda}{\|\tilde{\delta}_{1}\|_2}\right)$$

$$\text{Factor} = 1 - \frac{1.2}{2.449} \approx 1 - 0.49 = \mathbf{0.51}$$

즉, "이 변수 그룹은 원래 크기의 51%만 남기고 모두 똑같이 축소시켜라"라는 지시

#### 3단계: 비율 일괄 곱하기 (Scaling)

산출된 $0.51$이라는 하나의 비율(Scalar)을 군집 1, 2, 3의 원래 값에 똑같이 곱해줍니다. (일반 라쏘가 개별적으로 '빼기'를 했다면, 그룹 라쏘는 일괄적으로 '곱하기'를 합니다.)

- **군집 1:** $2.0 \times 0.51 = \mathbf{1.02}$
    
- **군집 2:** $-1.0 \times 0.51 = \mathbf{-0.51}$
    
- **군집 3:** $-1.0 \times 0.51 = \mathbf{-0.51}$

#### 최종 결과 비교: 파편화 vs 구조 보존

업데이트가 끝난 후, 모델이 갖게 되는 최종 파라미터는 다음과 같습니다.

- **업데이트된 군집 편차 ($\hat{\delta}_{1}$):** **$(1.02, -0.51, -0.51)$**
    

숫자의 절대적인 크기는 작아졌습니다. 하지만 가장 중요한 **군집 간의 비율을 확인해 보세요.**

$$1.02 : -0.51 : -0.51 \quad \longrightarrow \quad \mathbf{2 : -1 : -1}$$

일반 라쏘가 $\lambda=1.2$를 맞고 $(0.8, 0, 0)$이 되어 군집 2와 3의 정보를 완전히 상실했던 것과 달리, **그룹 라쏘는 페널티를 반영해도 원래 축이 가지고 있던 공간적 분리 비율($2 : -1 : -1$)을 왜곡하지 않고 완벽하게 보존**했습니다.
